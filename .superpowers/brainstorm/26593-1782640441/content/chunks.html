Heap Chunk Detector

Identifies heap allocation boundaries using MemoryInfo regions and pointer density

Approach

Heap chunks are inferred rather than directly parsed from the dump. Since minidumps lack allocation metadata (unlike a full memory dump), we use heuristics:

Phase 1 — Region partitioning: For each committed private heap region, identify boundaries: region start/end from MemoryInfo, pointer density valleys (gaps between dense clusters), and zero-filled runs.

Phase 2 — Density clustering: Graph nodes in the heap region are grouped by pointer density. Dense clusters with gaps between them suggest separate allocations.

Phase 3 — Boundary refinement: Align start/end to round allocation sizes (16-byte aligned typically for Windows heap).
