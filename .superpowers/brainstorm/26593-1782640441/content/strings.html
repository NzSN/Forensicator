String Detector

Scans AddressSpace for null-terminated byte sequences. Most reliable detection — validation is simple.

Detection algorithm

Phase 1 — Walk: Scan committed readable regions byte-by-byte. When a printable ASCII or valid UTF-16 lead byte is found, begin buffering.

Phase 2 — Buffer: Continue until null terminator or boundary. Reject if length < min_len (e.g. 4). Reject if non-printable chars exceed ratio threshold.

Phase 3 — Classify: Tag encoding (ASCII, UTF-16 LE, UTF-16 BE). Record VA + byte length + confidence.

StructString

va: u64
byte_len: usize
encoding: StringEncoding // Ascii | Utf16Le | Utf16Be
content: String // decoded readable text
confidence: f64 // 1.0 minus non-printable ratio

Parameters (tunable)

ParameterDefaultPurpose
min_len4Ignore noise
max_len65536Safety bound
max_nonprintable_ratio0.2Allow some control chars

Testing

Construct synthetic AddressSpace with known ASCII/UTF-16 strings at specific VAs, including edge cases (empty region, single char, max length, 100% non-printable, mixed encodings).
