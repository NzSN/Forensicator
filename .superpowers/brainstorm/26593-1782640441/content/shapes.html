Shape Clustering

Groups heap nodes by their outgoing edge pattern — structural type inference

What a "shape" is

A shape is the vector of (offset, target_class) for all outgoing edges of a node:

Node @ 0x1000: edges at +0x00→Heap, +0x08→Stack
Node @ 0x2000: edges at +0x00→Heap, +0x08→Stack
→ Same shape (ShapeGroup #1)

Node @ 0x3000: edges at +0x10→Image
→ Different shape (ShapeGroup #2) — likely a different C++ class

Algorithm

Phase 1 — Extract signatures: For each heap node in the graph, collect all outgoing edges. For each edge, compute the offset from the source VA (edge source VA − node VA). Build a signature: sorted vector of (offset, target_region_class).

Phase 2 — Hash and group: Hash each signature into a ShapeGroup. Nodes with identical signatures get the same group. This is O(n) with a HashMap.

Phase 3 — Rank: Sort groups by member count. Dominant shapes (most instances) are likely common classes.

ShapeGroup & ShapeSignature

pub struct ShapeSignature {
  edges: Vec<(u64, RegionClass)> // (offset, target_class) sorted by offset
}

pub struct ShapeGroup {
  id: usize,
  signature: ShapeSignature,
  member_count: usize,
  members: Vec<u64> // VAs of nodes in this group
}

pub struct ShapeClusters {
  groups: Vec<ShapeGroup> // sorted by member_count desc
}

StructureCatalog — final aggregation

pub struct StructureCatalog {
  strings: Vec<StructString>,
  vtables: Vec<StructVTable>,
  linked_lists: Vec<StructLinkedList>,
  arrays: Vec<StructArray>,
  chunks: Vec<StructChunk>,
  shape_clusters: ShapeClusters,
}
