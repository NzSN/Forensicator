List Detector

Finds linked lists — recurring edge patterns where each node points to the next at a consistent offset

What a linked list looks like in the graph

Singly-linked list (3 nodes):
  nodeA → (offset +0x08, conf 0.9) nodeB
  nodeB → (offset +0x08, conf 0.9) nodeC
  nodeC → null or absent

Key signature: same offset, same source region class, chain continues

Detection algorithm

Phase 1 — Pattern discovery: For each node in the graph that has ≥ 1 outgoing edge, compute the offset from the source VA to the next-pointer location. Group nodes by (offset, source_region_class). This identifies candidate "next pointer" offsets.

Phase 2 — Chain walking: Starting from unvisited nodes, follow edges at the discovered offset. Continue while the edge exists, confidence ≥ threshold, and the chain hasn't looped. Track visited nodes to avoid double-counting.

Phase 3 — Filter: Chains shorter than min_length are discarded. Chains that loop (circular lists) are tagged as circular.
