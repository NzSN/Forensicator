Array Detector

Finds sequential graph nodes with identical structure — arrays of objects or pointers

What an array looks like in the graph

Array of 4 elements @ 0x1000, stride 0x20:
  elem[0] @ 0x1000: out-degree=1, region=Heap, edge→0x2000
  elem[1] @ 0x1020: out-degree=1, region=Heap, edge→0x2100
  elem[2] @ 0x1040: out-degree=1, region=Heap, edge→null
  elem[3] @ 0x1060: out-degree=1, region=Heap, edge→0x2200

Key signature: constant stride, same out-degree, same region, sequential VAs

Detection algorithm

Phase 1 — Sort: Order all graph nodes by VA.

Phase 2 — Span: Walk nodes in VA order. When two consecutive nodes share the same out-degree, same region class, and VA difference ≤ max_stride, begin an array candidate.

Phase 3 — Extend: Continue as long as the next node matches the same pattern (degree, region, stride). End when stride changes or pattern breaks.

Phase 4 — Filter: Arrays shorter than min_length are discarded.
