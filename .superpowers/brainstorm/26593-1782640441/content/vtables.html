VTable Detector

Finds virtual method tables in module data — arrays of function pointers pointing into executable code

What a vtable looks like in memory

VTable @ 0x4001000 (in .rdata section):
  +0x00: 0x4000500 → module code (.text)
  +0x08: 0x4000600 → module code (.text)
  +0x10: 0x4000700 → module code (.text)
  +0x18: 0x4000800 → module code (.text)
  +0x20: 0x0000000 ← terminator (zero)

Detection algorithm

Phase 1 — Candidate scan: Walk module-data-typed committed regions. For each 8-byte-aligned word, check: is it a pointer into a known module's executable (.text) range?

Phase 2 — Run formation: If N consecutive candidates are found (N ≥ min_methods), with each pointing to executable code in the same module → candidate vtable.

Phase 3 — Termination check: End after a null entry or after a pointer to non-executable region. Validate minimum method count.

StructVTable

va: u64
method_count: usize
methods: Vec<u64> // list of function VAs
module_name: Option<String> // owner module if known
confidence: f64

Parameters

Testing

Construct module data region with known function pointers followed by zero terminator. Test: exact match, too few methods, interleaved non-code pointers, empty region.
