S3 — CLI & Testing

CLI — new subcommand

forensicator recover <dump.dmp> [flags]

Flags:
  --strings // run string detector only
  --vtables // run vtable detector only
  --lists // run list detector only
  --arrays // run array detector only
  --chunks // run chunk detector only
  --shapes // run shape clustering only
  --all // run everything (default)
  --json // JSON export of StructureCatalog
  --pattern <name> // S2 pattern for pointer scan (default: all presets)

Testing strategy

LayerApproach
stringsSynthetic AddressSpace with known ASCII/UTF-16 strings at specific VAs. Edge: empty region, non-printable sequences, max length.
vtablesSynthetic module data region with known function pointers + zero terminator. Edge: too few methods, interleaved garbage.
listsConstruct PointerGraph with known linked list pattern (nodes at known VAs, edges at fixed offsets). Edge: singleton, long chain, circular.
arraysConstruct PointerGraph with repeating node pattern at constant stride. Edge: varying stride, mixed degrees.
chunksSynthetic heap region with clustered nodes + gaps. Edge: single-node clusters, zero-filled ranges.
shapesGraph with multiple nodes sharing identical edge patterns + distinct patterns. Edge: empty graph, single-node graph.
integrationFull pipeline: synthetic dump → S1 parse → S2 scan+graph → S3 recover → verify catalog contents.

Dependencies

S3 depends on S2's PointerGraph + GraphQuery + S1's AddressSpace.
S3 adds 0 external dependencies. All detectors use existing infrastructure.
