S3 — Detector Trait & StructureCatalog

Each recovery task is a module implementing StructureDetector. Output unified into StructureCatalog.

pub trait StructureDetector {
  type Item;
  fn name(&self) -> &str;
  fn detect(
    &self,
    space: &AddressSpace,
    graph: &PointerGraph,
    query: &GraphQuery,
  ) -> Vec<Self::Item>;
}

5 detector modules

strings/

Scans AddressSpace for null-terminated ASCII/UTF-16 byte sequences. Output: StructString { va, len, encoding }

vtables/

Finds arrays of function pointers in module data — each entry points to executable code. Output: StructVTable { va, method_count, methods: Vec<u64> }

lists/

Uses GraphQuery to find consistent edge patterns — node → next at fixed offset. Output: StructLinkedList { head_va, length, stride, next_offset }

arrays/

Finds sequences of graph nodes with identical structure and sequential VAs. Output: StructArray { start_va, element_size, count, element_type }

chunks/

Identifies heap allocation boundaries via MemoryInfo + pointer density. Output: StructChunk { va_start, size, is_free }

Unified output: StructureCatalog

pub struct StructureCatalog {
  strings: Vec<StructString>,
  vtables: Vec<StructVTable>,
  linked_lists: Vec<StructLinkedList>,
  arrays: Vec<StructArray>,
  chunks: Vec<StructChunk>,
}

// Built by running all detectors:
pub fn recover_all(space, graph, query) -> StructureCatalog
