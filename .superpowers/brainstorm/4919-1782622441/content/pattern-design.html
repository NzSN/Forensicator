PointerPattern — Three Independent Layers

Byte-level matching • Source context • Target context. All independent, all optional.

Layer 1: ValueMatcher — what bytes look like a pointer
Tests the 8-byte value itself, before any lookup

Layer 2: Source filter — where the value was found
Stack, heap, register, module data, any committed

Layer 3: Target filter — what does it point to
Image, stack, heap, mapped, any readable — requires AddressSpace lookup

Layer 1 — ValueMatchers

Composable predicates on raw 8-byte values. All must match for a candidate to pass.

AlignedTo(bytes: 8)

LSB = 0 • typical 8-byte pointer

BitMask { mask, expected }

Arbitrary bit pattern check

CanonicalX64

Bits 48–63 match bit 47

InRange(lo, hi)

User-space filter

Modulo(div, rem)

4-byte alignment in 64=bit dump

KnownVA(addr)

Matches exact known address

BitZero(n) / BitOne(n)

Single bit test at position n

MatchSize(sz: 4 | 8)

4-byte vs 8-byte pointer width

Example: user's LSB-0 pattern

PointerPattern {
  name: "aligned_x64_pointers",
  value_matchers: vec![AlignedTo(8), CanonicalX64],
  source: SourceContext::AnyCommitted,
  target: TargetContext::AnyReadable,
  min_confidence: 0.7,
  ..Default::default()
}

Value matchers are AND-ed — all must pass. If any single source/target field is set, it narrows. Empty = unfiltered.

Confidence scoring

When a candidate passes a pattern's value matchers, it gets a base score based on how much evidence supports it:

Evidence+Score
Value is aligned (LSBs = 0)+0.15
Value is canonical x64 address+0.20
Target VA is readable+committed+0.25
Target falls inside a known module+0.15
Value lives in a register+0.15
Value lives at stack offset used by call convention+0.10
