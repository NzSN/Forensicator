S2 Module Layout — Tiered Pipeline

4 modules in forensicator-core/src/. Each testable independently.

forensicator-core/src/
  pattern/ ← user-definable pointer patterns
  scan/ ← word scanner + classifier + root extractor
  graph/ ← pointer graph construction (nodes + edges)
  query/ ← traversal, filtering, statistics, dot export
  model.rs ← extended — new S2 types (Pointer, Edge, Node, RootSet)
  space.rs ← existing — scanner reads AddressSpace

pattern → scan → graph → query
        ↘        ↓
         model   space

Data flow

PointerPattern[]  →  Scan → Candidate[]  →  Build → PointerGraph  →  GraphQuery
(user config) (reads AddressSpace) (nodes+edges+roots) (API for S3/CLI)

What S3 imports

S3 only depends on: PointerGraph (from graph), GraphQuery (from query), and S2 model types.
Pattern definitions and scanner internals stay hidden.
