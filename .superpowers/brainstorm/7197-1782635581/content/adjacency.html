adj_out & adj_in — Dual Adjacency Index

Two parallel vecs indexed by node. Each stores edge indices, not raw data.

adj_out

"What does this node point to?"

adj_out[n0] = [e0, e1]    n0 → n1, n0 → n2
adj_out[n1] = [e2]       n1 → n2
adj_out[n2] = []          n2 points nowhere

adj_in

"Who points to this node?"

adj_in[n0] = []          nothing points to n0
adj_in[n1] = [e0]       n0 → n1
adj_in[n2] = [e1, e2]    n0 → n2, n1 → n2
    
  n0 ───── n2
   └───n1───┘

Why both?

S3: "find all pointers to heap object X"adj_in[x] — O(1)
S3: "trace everything reachable from root R"BFS over adj_out — O(|V|+|E|)
S3: "is there a back-pointer from Y to X?"check adj_in[y] for edge with from=x — O(out_degree)
S4: "what's the root set for this subgraph?"BFS backwards over adj_in — O(|V|+|E|)

Without adj_in, reverse queries would be O(|E|) full scan. Edge indices stored (not copies) — memory stays compact.
