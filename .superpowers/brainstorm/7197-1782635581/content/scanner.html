Scan Module

Iterates AddressSpace, applies patterns, extracts root sets, emits ranked candidates

Input / Output

Input:
  &AddressSpace   // from S1 space.rs
  &Dump            // registers, modules, stacks for root extraction
  &[PointerPattern]  // user-configured patterns
Output:
  ScanResult { candidates: Vec<CandidatePointer>, roots: Vec<Root> }

CandidatePointer

source_va: u64       // where the pointer value lives in memory
value: u64          // the raw 8-byte value read
target_va: u64      // where it points (== value if valid VA)
source_ctx: SourceContext  // stack/heap/register/module
target_ctx: TargetContext  // what region it points into
confidence: f64      // 0.0..1.0
matched_by: Vec<String> // which pattern(s) matched
evidence: Vec<String> // human-readable: "aligned", "canonical", "target is image"

Scan strategy

Two-phase scan for performance on large dumps:

Phase 1 — Value scan: Linear walk of committed regions at match_size stride. For each word, test all patterns' value matchers. Keep passing candidates in a small buffer. Value-only tests are cheap (no AddressSpace lookup).

Phase 2 — Target resolution: For each value-passing candidate, do the AddressSpace lookup (target classification). Apply source+target filters, compute confidence. This phase is I/O-heavy so candidates are batched.

Root extraction

Roots are the starting points for graph traversal. Three sources:

Register roots

Every non-zero register in each thread's CONTEXT. RIP, RSP, RBP, plus GPRs. Small set (~dozens).

Stack roots

Every candidate pointer found in thread stack regions. Typically hundreds to thousands.

Module data roots

Pointers in module .data/.rdata/.bss sections. Globals, vtables, function pointers. Large set.

Boundaries

Scanner does NOT build edges or resolve duplicate pointers. That's the graph module's job. Scanner produces a flat list of candidates + root set. No deduplication, no graph construction.
