Query Module

Traversal, filtering, statistics, and export API on PointerGraph

GraphQuery — the public handle

graph: &PointerGraph
config: QueryConfig

QueryConfig carries defaults: min_confidence, max_depth, region_filters. Queries can override per-call.

Traversal API

MethodReturnsS3 use case
is_reachable(va)boolDead object detection
path_to_root(va)Vec<EdgePath>Ownership chains
all_paths(from, to, max_depth)Vec<Vec<EdgePath>>Multiple reference paths
reachable_from(va, pred)Vec<NodeIndex>Subgraph extraction
who_points_to(va, pred)Vec<NodeIndex>Reverse references
neighbors(va)(Vec<NodeIndex>, Vec<NodeIndex>)In/out neighborhood

Each traversal accepts an optional EdgePredicate to filter by confidence, depth, source/target region class.

EdgePredicate — composable filters

min_confidence: f64
max_confidence: f64
source_region: Option<RegionClass>
target_region: Option<RegionClass>
max_depth: Option<usize>
matched_by_pattern: Option<String>

largest_component()Size of largest weakly connected component

Export

to_dot(filter) — Graphviz DOT output. S3/S4 or CLI use this for debugging. Edges colored by confidence (green=high, yellow=mid, red=low). Nodes shaped by region class (box=image, oval=heap, diamond=stack).

to_json(filter) — Machine-readable export. Nodes with VA+class, edges with confidence+evidence.

TLA+ spec

Extension of PointerGraph.tla: models BFS reachability as transitive closure of edges above confidence threshold. Invariants: is_reachable is transitive, path_to_root paths are valid edge sequences, who_points_to is the inverse of edge direction.
